SCARL Rate-Limiting Project

SCARL is a smart contract API rate-limiting project that combines on-chain policy with off-chain gateway enforcement. The system uses token buckets for burst control, fixed window counters for quota enforcement, and adaptive logic to adjust limits based on tenant behavior and system load. This project helped me communicate distributed systems concepts through architecture diagrams, error responses, and technical explanations. It illustrates my ability to work with complex backend systems while still making the design understandable to others. The project also connects directly to my interest in scalable software systems and practical technical infrastructure.
